Learning Model

Atomize → Loop → Master → Stack → Expand

No modes. No settings. No quizzes. Just loop a line until it's yours, then chain it with others. Like building with Lego bricks.

Step 1

🎵 Pick a Line

Tap any line on the checklist. Card slides up. Audio starts looping.

→
Step 2

🔁 Loop It

Audio plays the line, ~1s gap, plays again. You sing along. Translation visible (tap to hide).

→
Step 3

✓ Master It

Tap "Mastered." Card slides down. Checkmark appears. Line is yours.

→
Step 4

🧱 Stack It

New stack buttons appear: Play 1–2, Play 1–3. Continuous audio, looping.

→
Step 5

📈 Expand

Keep mastering lines. Stack grows. Eventually you own the whole song.

Why this works: Memory hates big jumps. It loves incremental expansion. If you can sing one line comfortably, adding one more feels 30% harder, not 100% harder. That's how you get from nothing to a full verse without overwhelm.

Interaction Design

Card States & Transitions

The slide-up card has three states. The transitions are simple — there's almost nothing to learn.

State: Looping

🔊 Audio Playing

Line audio plays, ~1s silence, loops again. Loop counter increments. Translation visible (or hidden if toggled).

→ Tap Mastered: slide down, mark ✓

→ Tap Pause: enter paused state

→ Swipe down: dismiss, go to checklist

⇄
State: Paused

⏸ Frozen

Audio stopped. Card stays. User studies the line or takes a break. Pause button becomes Resume.

→ Tap Mastered: slide down, mark ✓

→ Tap Resume: continue looping

→ Swipe down: dismiss

→
State: Mastered

✓ Done

Brief flash of green. Card slides down. Checkmark appears on checklist. Stack buttons update if applicable.

→ Automatic: returns to checklist

What's NOT here: No Next button (swipe down and pick from checklist). No Flow/Study mode toggle (everything loops). No settings gear. No countdown timers. The card does one thing: loop a line until you master it.
Architecture

Technical Stack

Same proven stack. Dramatically simpler feature surface.

📱

Alpine.js Frontend

Reactive state · No build step · 15KB · Slide-up card with CSS transitions

🐍

FastAPI Backend

REST API · Async · Song/lyrics/progress endpoints

🗄️

SQLite

Zero config · Songs, lines, mastery status

🤖

Claude API

Full-song translation · JSON mapping

🎙️

Whisper

Auto-timestamps · Fuzzy alignment · Local CPU

🎵

Audio

HTML5 Audio (MP3) · Spotify SDK · YouTube (v2)

☁️

DigitalOcean

Droplet · nginx · GitHub deploys

What changed from v1.x: The backend barely changes. The frontend is dramatically simpler — one main screen (checklist) with a slide-up card overlay. Study Mode, Flow Mode, configurable pauses, countdown overlays, three-button card layout, segment view as separate screen — all gone. Replaced by: loop + master + stack.
